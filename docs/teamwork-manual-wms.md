## 📌 ภาพรวมระบบ WMS

### ระบบ WMS คืออะไร?

**VNIX Warehouse Management System (WMS)** คือระบบบริหารจัดการคลังสินค้าที่เชื่อมโยง:
- 📱 **ออเดอร์จากหลายแพลตฟอร์ม** (Shopee, Lazada, TikTok)
- 📦 **การหยิบสินค้าในคลัง** (Picking)
- 🚚 **การส่งมอบให้ขนส่ง** (Dispatch)

### เป้าหมายของระบบ

✅ **ลดเวลาในการประมวลผลออเดอร์** จากหลายชั่วโมง → 30-60 นาที
✅ **ลดความผิดพลาดในการหยิบสินค้า** ด้วยระบบสแกน QR Code
✅ **ติดตามสต็อกแบบ Real-time** รู้ทันทีว่ามีของเหลือกี่ชิ้น
✅ **จัดการ Shortage อย่างเป็นระบบ** ไม่มีของหายหรือลืม
✅ **ส่งของทัน SLA** ตามเวลาที่แต่ละแพลตฟอร์มกำหนด

### คำศัพท์สำคัญที่ต้องรู้

| คำศัพท์ | คำอธิบาย | ตัวอย่าง |
|---------|---------|----------|
| **SKU** | รหัสสินค้า (Stock Keeping Unit) | SKU-001, SKU-002 |
| **Batch** | ชุดงานหยิบสินค้า (รวมหลายออเดอร์) | B-20251119-001 |
| **Run Number** | รอบที่สร้าง Batch ในวันเดียวกัน | R1, R2, R3... |
| **Picking** | การหยิบสินค้าตาม Batch | |
| **Packing** | การแพ็คสินค้าเพื่อส่งมอบ | |
| **Shortage** | สินค้าขาด (หยิบไม่ครบ) | ขาด 5 ชิ้น |
| **Handover Code** | รหัสส่งมอบให้ขนส่ง | H-20251119-001 |
| **SLA** | เวลาที่ต้องส่งของตามที่แพลตฟอร์มกำหนด | Shopee: 2 วัน, TikTok: 1 วัน |
| **Progress** | ความคืบหน้าของ Batch (%) | 75% = หยิบไป 75 ชิ้นจาก 100 ชิ้น |

### 🛒 ทีม Online (Online Team)

**หน้าที่หลัก:**
- 📥 นำเข้าข้อมูล (Import Orders, Products, Stock)
- 📦 สร้าง Batch สำหรับการหยิบของ
- 📊 ติดตามความคืบหน้า (Progress Monitoring)
- ⚠️ จัดการ Shortage Queue
- 📄 พิมพ์เอกสารและ Export รายงาน

**บุคคลในทีม:**
- **Online Admin:** ทำได้ทุกอย่าง + จัดการผู้ใช้
- **Online Staff:** สร้าง Batch และติดตาม Progress

### 📦 ทีม Picker (พนักงานหยิบของ)

**หน้าที่หลัก:**
- 📲 รับงาน Batch (Scan Batch)
- 🔍 หยิบสินค้า (Scan SKU)
- ⚠️ บันทึก Shortage (ถ้าสินค้าไม่พอ)

**เครื่องมือ:**
- มือถือ/แท็บเล็ต สำหรับสแกน QR Code
- ใบงาน Picking List (พิมพ์จากทีม Online)

### 📋 ทีม Packer (พนักงานแพ็คและส่งมอบ)

**หน้าที่หลัก:**
- 📦 แพ็คสินค้าที่หยิบเสร็จแล้ว
- 🚚 สแกนส่งมอบให้ขนส่ง (Scan Handover)
- 🖨️ พิมพ์ใบส่งมอบ (Handover Slip)

**เครื่องมือ:**
- มือถือ/แท็บเล็ต สำหรับสแกน
- เครื่องพิมพ์ใบส่งมอบ

### 📝 การทำงานแต่ละขั้นตอน (Step-by-Step)

#### 🔹 Step 1: Import Orders (นำเข้าออเดอร์)

**เวลาที่ทำ:** เช้า 08:00-09:00 (หรือเมื่อมีออเดอร์ใหม่)

**วัตถุประสงค์:** นำเข้าออเดอร์จากแพลตฟอร์มต่างๆ เข้าสู่ระบบ

**ขั้นตอน:**

1. **เข้าหน้า Import Orders**
   - คลิกเมนู `นำเข้า` → `Import Orders`

2. **เตรียมไฟล์ Excel**
   - ไฟล์ต้องมีคอลัมน์ที่จำเป็น:
     - `Order Number` (เลขที่ออเดอร์)
     - `Platform` (Shopee/Lazada/TikTok)
     - `SKU` (รหัสสินค้า)
     - `Quantity` (จำนวน)
     - `Shop Name` (ชื่อร้าน)
     - `Logistic` (ขนส่ง - optional)
     - `Due Date` (วันที่ต้องส่ง - optional, ถ้าไม่มีจะคำนวณอัตโนมัติ)

   **ตัวอย่างไฟล์ Excel:**
   ```
   Order Number | Platform | SKU     | Quantity | Shop Name | Logistic | Due Date
   SO123456     | Shopee   | SKU-001 | 2        | Shop A    | SPX      | 2025-11-21
   SO123457     | Lazada   | SKU-002 | 5        | Shop B    | Flash    | 2025-11-22
   SO123458     | TikTok   | SKU-003 | 1        | Shop C    | LEX      | 2025-11-20
   ```

3. **อัพโหลดไฟล์**
   - คลิก `เลือกไฟล์` → เลือกไฟล์ Excel → คลิก `Import`

4. **ตรวจสอบผลลัพธ์**
   - ✅ **สำเร็จ:** จะแสดงข้อความ "Import สำเร็จ X รายการ"
   - ❌ **ผิดพลาด:** จะแสดงข้อความ error พร้อมบรรทัดที่ผิด

5. **ตรวจสอบ Dashboard**
   - ไปที่ `Dashboard` → ดูว่าออเดอร์ถูก classify เป็น:
     - 🟢 **READY_ACCEPT** - มีสต็อกพอ พร้อมสร้าง Batch
     - 🟡 **LOW_STOCK** - สต็อกน้อย อาจหยิบได้บางส่วน
     - 🔴 **SHORTAGE** - ไม่มีสต็อก ต้องรอของเข้า

**💡 เคล็ดลับ:**
- Import ข้อมูลทุกเช้าก่อน 09:00 น. เพื่อให้ทีมคลังมีงานทำตั้งแต่เช้า
- ตรวจสอบ SKU ว่าตรงกับฐานข้อมูลหรือไม่ก่อน Import
- ถ้ามีออเดอร์เยอะ (500+) ควรแบ่ง Import เป็นหลายไฟล์

**⚠️ สิ่งที่ต้องระวัง:**
- ❌ **ห้าม Import ข้อมูลซ้ำ** → จะสร้างออเดอร์ซ้ำ
- ❌ **ห้าม Import SKU ที่ไม่มีในระบบ** → จะเกิด error หรือสร้างออเดอร์ที่หยิบไม่ได้
- ⚠️ **ถ้า Platform ใส่ผิด** (เช่น พิมพ์ shopee แทน Shopee) → ระบบจะแปลงให้อัตโนมัติ แต่ควรตรวจสอบก่อน

**🆘 ถ้าเกิดปัญหา:**
- **Import ผิด:** ลบออเดอร์ที่ผิด → Import ใหม่
- **SKU ไม่มีในระบบ:** Import Products ก่อน → แล้วค่อย Import Orders
- **ไฟล์ Excel เปิดไม่ได้:** ตรวจสอบว่าเป็น .xlsx และไม่มี password

---

#### 🔹 Step 2: Import Products (นำเข้าสินค้า)

**เวลาที่ทำ:** เมื่อมีสินค้าใหม่ หรือต้องการอัพเดตข้อมูล

**วัตถุประสงค์:** เพิ่มสินค้าใหม่เข้าระบบ

**ขั้นตอน:**

1. **เข้าหน้า Import Products**
   - คลิกเมนู `นำเข้า` → `Import สินค้า`

2. **เตรียมไฟล์ Excel**
   - คอลัมน์ที่จำเป็น:
     - `SKU` (รหัสสินค้า - ต้องไม่ซ้ำ)
     - `Description` (รายละเอียดสินค้า)
     - `Platform` (optional)

   **ตัวอย่าง:**
   ```
   SKU     | Description           | Platform
   SKU-001 | iPhone 15 Pro Max     | Shopee
   SKU-002 | Samsung Galaxy S24    | Lazada
   SKU-003 | AirPods Pro 2         | TikTok
   ```

3. **อัพโหลดและตรวจสอบ**
   - คลิก `Import` → ดูผลลัพธ์

**⚠️ สิ่งที่ต้องระวัง:**
- ❌ **SKU ซ้ำ** → ระบบจะไม่ Import หรืออัพเดตข้อมูลเดิม
- ⚠️ **Description ว่างเปล่า** → ควรใส่เพื่อให้ทีมคลังรู้ว่าเป็นสินค้าอะไร

---

#### 🔹 Step 3: Import Stock (นำเข้าสต็อก)

**เวลาที่ทำ:** เมื่อมีของเข้าใหม่ หรือต้องการอัพเดตจำนวน

**วัตถุประสงค์:** อัพเดตจำนวนสต็อกในคลัง

**ขั้นตอน:**

1. **เข้าหน้า Import Stock**
   - คลิกเมนู `นำเข้า` → `Import สต็อก`

2. **เตรียมไฟล์ Excel**
   - คอลัมน์ที่จำเป็น:
     - `SKU`
     - `Shop Name` (ร้านที่เก็บสต็อก)
     - `Shelf` (ชั้นวางของ เช่น A-01, B-05)
     - `Quantity` (จำนวน)

   **ตัวอย่าง:**
   ```
   SKU     | Shop Name | Shelf | Quantity
   SKU-001 | Shop A    | A-01  | 100
   SKU-002 | Shop B    | B-05  | 50
   SKU-003 | Shop A    | C-10  | 25
   ```

3. **Import และตรวจสอบ**
   - ระบบจะอัพเดตสต็อกตาม SKU + Shop + Shelf
   - ถ้ามีอยู่แล้ว → **เขียนทับ** (ไม่ได้บวกเพิ่ม)

**💡 เคล็ดลับ:**
- Import Stock ทุกครั้งที่มีของเข้า เพื่อให้ข้อมูลสต็อกถูกต้อง
- ใช้ Shelf เป็นตัวบอกตำแหน่งของในคลัง ให้ Picker หาของง่าย

**⚠️ สิ่งที่ต้องระวัง:**
- ⚠️ **Import Stock จะเขียนทับ** → ถ้าใส่ Quantity = 10 แล้วมีอยู่ 50 ก็จะกลายเป็น 10
- ⚠️ **Shelf ต้องตรงกับที่ตั้งจริง** → ไม่งั้น Picker จะหาของไม่เจอ

---

#### 🔹 Step 4: Create Batch (สร้างชุดงานหยิบของ)

**เวลาที่ทำ:** หลัง Import Orders เรียบร้อยแล้ว (09:00-10:00)

**วัตถุประสงค์:** รวมออเดอร์ที่พร้อมส่งเป็น Batch เพื่อให้ Picker หยิบ

**ขั้นตอน:**

1. **เข้าหน้า Create Batch**
   - คลิกเมนู `Batch Management` → `สร้าง Batch ใหม่`

2. **เลือกเงื่อนไข**
   - **Platform:** เลือก Shopee, Lazada, หรือ TikTok (ทีละแพลตฟอร์ม)
   - **วันที่:** เลือกช่วงวันที่ที่ต้องการสร้าง Batch
   - **ประเภทออเดอร์:** มักจะเลือก "READY_ACCEPT" เท่านั้น

3. **ตรวจสอบออเดอร์**
   - ระบบจะแสดงรายการออเดอร์ที่ตรงเงื่อนไข
   - ตรวจสอบ:
     - ✅ จำนวนออเดอร์
     - ✅ จำนวน SKU ที่ต้องหยิบ
     - ✅ Carrier Summary (SPX, Flash, LEX, J&T, Other)
     - ✅ Shop Summary

4. **สร้าง Batch**
   - คลิก `สร้าง Batch`
   - ระบบจะสร้าง:
     - **Batch ID:** เช่น `B-20251119-001` (B-วันที่-เลขรัน)
     - **Run Number:** เช่น `R1`, `R2`, `R3` (รอบที่สร้างในวันเดียวกัน)
     - **QR Code:** สำหรับ Picker สแกนรับงาน

5. **พิมพ์เอกสาร**
   หลังสร้าง Batch เสร็จ ควรพิมพ์เอกสารเหล่านี้:

   **📄 Picking List** (ใบงานหยิบของ)
   - คลิก `พิมพ์ใบงานคลัง`
   - มีรายการ SKU ทั้งหมด + จำนวน + ชั้นวางของ (Shelf)
   - มี QR Code ของ Batch สำหรับสแกน

   **🏷️ SKU QR Code** (สติกเกอร์ SKU)
   - คลิก `พิมพ์ QR Code SKU`
   - ติดที่ชั้นวางของ เพื่อให้ Picker สแกนขณะหยิบ

6. **ส่งงานให้ทีมคลัง**
   - ส่งใบงาน Picking List ให้ทีม Picker
   - แจ้งว่ามี Batch ใหม่พร้อมหยิบ

**💡 เคล็ดลับ:**
- **สร้าง Batch แยกตาม Platform** → เพราะแต่ละ Platform มี SLA ต่างกัน
  - TikTok: 1 วัน (ด่วนที่สุด)
  - Shopee: 2 วัน
  - Lazada: 3 วัน
- **สร้างหลาย Batch ในวันเดียวได้** → จะได้ R1, R2, R3... เพื่อแบ่งงานให้หลาย Picker
- **ตรวจสอบ Carrier Summary** → ถ้า Carrier เดียวเยอะมาก อาจจะแยก Batch

**⚠️ สิ่งที่ต้องระวัง:**
- ❌ **ห้ามสร้าง Batch จากออเดอร์ที่เป็น SHORTAGE** → หยิบไม่ได้เพราะไม่มีของ
- ❌ **สร้าง Batch แล้วไม่สามารถแก้ไข Platform ได้** → ถ้าผิดต้องลบและสร้างใหม่
- ⚠️ **ลืมพิมพ์ Picking List** → ทีม Picker จะไม่รู้ว่าต้องหยิบอะไร
- ⚠️ **Run Number จะเพิ่มอัตโนมัติ** → ถ้าสร้าง Batch วันเดียวกัน จะได้ R1, R2, R3...

**🆘 ถ้าเกิดปัญหา:**
- **สร้าง Batch ผิด Platform:** ลบ Batch (Admin เท่านั้น) → สร้างใหม่
- **ออเดอร์ไม่แสดงในหน้าสร้าง Batch:** ตรวจสอบว่าเป็น READY_ACCEPT หรือยัง (ดูที่ Dashboard)
- **สร้างแล้วไม่มี QR Code:** Refresh หน้า → ดู Batch Detail อีกครั้ง

#### 🔹 Step 5: Monitor Progress (ติดตามความคืบหน้า)

**เวลาที่ทำ:** ตลอดทั้งวัน (10:00-17:00)

**วัตถุประสงค์:** ติดตามว่า Picker หยิบของไปถึงไหนแล้ว

**ขั้นตอน:**

1. **ดู Batch List**
   - คลิกเมนู `Batch Management` → `รายการ Batch`
   - จะเห็นตาราง Batch ทั้งหมด พร้อม:
     - **Batch ID**
     - **Platform**
     - **Run Number**
     - **จำนวนออเดอร์**
     - **Progress (%)** ← สำคัญ!
     - **สถานะส่งมอบ**

2. **ตรวจสอบ Progress Bar**
   - 🟢 **100%** → หยิบครบแล้ว พร้อมส่งมอบ
   - 🔵 **50-99%** → กำลังหยิบอยู่
   - 🟡 **< 50%** → เพิ่งเริ่ม หรือมีปัญหา
   - 🔴 **0%** → ยังไม่มี Picker รับงาน

3. **คลิกดู Batch Detail**
   - คลิกที่ Batch ID เพื่อดูรายละเอียด
   - จะเห็น:
     - **SKU Progress Table** → แต่ละ SKU หยิบไปกี่ชิ้นแล้ว
     - **สถานะแต่ละ SKU:**
       - ✅ **Completed** (สีเขียว) → หยิบครบ
       - ⏳ **Picking** (สีเหลือง) → หยิบไม่ครบ
       - ❌ **Shortage** (สีแดง) → มีของขาด

4. **ติดตามแบบ Real-time**
   - เมื่อ Picker สแกน SKU → Progress จะอัพเดททันที
   - Refresh หน้า (F5) เพื่อดูข้อมูลล่าสุด

**💡 เคล็ดลับ:**
- ตรวจสอบ Progress ทุก 1-2 ชั่วโมง เพื่อดูว่ามีปัญหาไหม
- ถ้า Progress ติดอยู่ที่เดิมนาน → โทรถาม Picker ว่าเกิดอะไรขึ้น
- ถ้ามี Shortage เยอะ → ดูที่ Shortage Queue และแก้ไข

**⚠️ สิ่งที่ต้องระวัง:**
- ⚠️ **Progress อาจไม่ถึง 100% ถ้ามี Shortage** → ต้อง Resolve Shortage ก่อน
- ⚠️ **Picker หยิบผิด SKU** → Progress จะไม่อัพเดท ต้องแจ้งให้แก้ไข

#### 🔹 Step 6: Manage Shortage Queue (จัดการสินค้าขาด)

**เวลาที่ทำ:** เมื่อ Picker แจ้ง Shortage (บ่าย 15:00-16:00)

**วัตถุประสงค์:** แก้ไขปัญหาสินค้าขาด เพื่อให้ส่งของได้

**ขั้นตอน:**

1. **เข้าหน้า Shortage Queue**
   - คลิกเมนู `Batch Management` → `Shortage Queue`

2. **ดูรายการ Shortage**
   - ตารางจะแสดง:
     - **Order ID**
     - **SKU** (สินค้าที่ขาด)
     - **Quantity Shortage** (จำนวนที่ขาด)
     - **Reason** (เหตุผล: สต็อกหมด, สินค้าเสีย, อื่นๆ)
     - **Status** (สถานะ: waiting_stock, cancelled, replaced, resolved)
     - **Urgency** (ความเร่งด่วน: high, medium, low)

3. **เลือกวิธีแก้ไข**

   **🔄 กรณีที่ 1: รอสต็อกเข้า (waiting_stock)**
   - ถ้ามีของเข้าในอีก 1-2 วัน
   - เลือก Status = `waiting_stock`
   - เมื่อของเข้า → Import Stock ใหม่ → แจ้ง Picker ให้หยิบเพิ่ม

   **❌ กรณีที่ 2: ยกเลิกออเดอร์ (cancelled)**
   - ถ้าลูกค้ายกเลิก หรือไม่มีของแน่นอน
   - เลือก Status = `cancelled`
   - Batch สามารถส่งมอบได้โดยไม่ต้องหยิบ SKU นี้

   **🔁 กรณีที่ 3: แทน SKU (replaced)**
   - ถ้ามีสินค้าทดแทนได้
   - เลือก Status = `replaced`
   - ระบุ SKU ใหม่ที่จะแทน

   **✅ กรณีที่ 4: จัดการเรียบร้อย (resolved)**
   - ถ้าพบของที่หายไป หรือนับสต็อกผิด
   - เลือก Status = `resolved`
   - แจ้ง Picker ให้หยิบใหม่

4. **บันทึก**
   - คลิก `บันทึก`
   - Shortage จะถูกอัพเดทสถานะ
   - ถ้า Resolve แล้ว → Batch Progress จะเป็น 100%

**💡 เคล็ดลับ:**
- ควร Resolve Shortage ให้เร็วที่สุด เพื่อไม่ให้เสีย SLA
- ถ้ามี Shortage เยอะ → ตรวจสอบสต็อกว่าถูกต้องหรือไม่
- ใช้ Urgency เพื่อ prioritize Shortage ที่เร่งด่วน

**⚠️ สิ่งที่ต้องระวัง:**
- ⚠️ **ถ้า Cancel Shortage แล้วลืมบอกลูกค้า** → ลูกค้าจะไม่ได้รับของ
- ⚠️ **Replace SKU ผิด** → ลูกค้าได้ของไม่ตรงกับที่สั่ง

#### 🔹 Step 1: รับงาน Batch (Scan Batch)

**เวลาที่ทำ:** เริ่มงาน (10:00) หรือเมื่อพร้อมรับงานใหม่

**วัตถุประสงค์:** รับ Batch จากทีม Online เพื่อเริ่มหยิบสินค้า

**ขั้นตอน:**

1. **รับใบงาน Picking List**
   - ทีม Online จะส่งใบงานให้ (พิมพ์จากระบบ)
   - ใบงานจะมี:
     - Batch ID
     - QR Code
     - รายการ SKU + จำนวน + Shelf

2. **เข้าหน้า Scan Batch**
   - เปิดมือถือ/แท็บเล็ต
   - เข้าเมนู `สแกนคลัง` → `1. สแกนรับงาน (Batch)`

3. **สแกน QR Code**
   - **วิธีที่ 1:** คลิก `📷 สแกน Batch QR Code` → สแกน QR บนใบงาน
   - **วิธีที่ 2:** เลือกจาก `Quick Assign` (ถ้าระบบตั้งค่าไว้)
   - **วิธีที่ 3:** พิมพ์ Batch ID ด้วยตัวเอง (ถ้าสแกนไม่ได้)

4. **ยืนยันรับงาน**
   - ระบบจะแสดงรายละเอียด Batch:
     - จำนวนออเดอร์
     - จำนวน SKU
     - รายการ SKU ที่ต้องหยิบ
   - คลิก `✅ รับงาน`

5. **เริ่มหยิบ**
   - ระบบจะบันทึกว่าคุณรับงาน Batch นี้แล้ว
   - ไปที่ขั้นตอนถัดไป: Scan SKU

**💡 เคล็ดลับ:**
- รับ Batch ทีละ 1 Batch เพื่อไม่ให้งานซ้อน
- ดูใบงาน Picking List เพื่อวางแผนเส้นทางหยิบของ (Shelf A ก่อน → B → C)
- ถ้ามี Batch หลายตัว ให้เลือก Batch ที่ SLA ใกล้ก่อน (ดูที่ Due Date)

**⚠️ สิ่งที่ต้องระวัง:**
- ❌ **สแกน Batch ผิด** → รับงานคนอื่น, ทำให้สับสน
  - **วิธีแก้:** ยกเลิกและสแกนใหม่
- ⚠️ **รับหลาย Batch พร้อมกัน** → อาจจำไม่ได้ว่ากำลังทำ Batch ไหน
  - **วิธีแก้:** ทำให้เสร็จทีละ Batch

**🆘 ถ้าเกิดปัญหา:**
- **QR Code สแกนไม่ได้:** พิมพ์ Batch ID ด้วยตัวเอง
- **Batch ไม่แสดงใน Quick Assign:** บอกทีม Online ให้ตรวจสอบว่าสร้าง Batch แล้วหรือยัง

---

#### 🔹 Step 2: หยิบสินค้า (Scan SKU)

**เวลาที่ทำ:** หลังรับงาน Batch แล้ว (ตลอดวัน)

**วัตถุประสงค์:** หยิบสินค้าตามรายการใน Batch

**ขั้นตอน:**

1. **เข้าหน้า Scan SKU**
   - เปิดมือถือ/แท็บเล็ต
   - เข้าเมนู `สแกนคลัง` → `2. สแกนหยิบของ (SKU)`

2. **เลือก Batch**
   - เลือก Batch ที่เพิ่งรับมา
   - หรือใช้ Quick Assign

3. **หา SKU ตามใบงาน**
   - ดูใบงาน Picking List → หา SKU
   - ดู Shelf (เช่น A-01, B-05) เพื่อหาตำแหน่งของ

4. **สแกน SKU QR Code**
   - **วิธีที่ 1:** คลิก `📷 สแกน SKU` → สแกน QR บนชั้นวางของ
   - **วิธีที่ 2:** พิมพ์ SKU ด้วยตัวเอง

5. **ระบุจำนวนที่หยิบได้**
   - ระบบจะแสดง:
     - **ต้องการ:** 10 ชิ้น
     - **หยิบได้:** ___ ชิ้น (ให้คุณกรอก)

   **กรณีที่ 1: หยิบครบ (ของพอ)**
   - พิมพ์จำนวนที่หยิบได้ = ต้องการ (เช่น 10 = 10)
   - คลิก `✅ บันทึก`
   - ระบบจะแสดง "บันทึกสำเร็จ"
   - SKU นี้จะเป็น "Completed" (สีเขียว)

   **กรณีที่ 2: หยิบไม่ครบ (ของไม่พอ)**
   - พิมพ์จำนวนที่หยิบได้ < ต้องการ (เช่น หยิบได้ 5 แต่ต้องการ 10)
   - คลิก `⚠️ Mark Shortage` (ไม่ใช่บันทึก)
   - ไปที่ Step 3: Mark Shortage

6. **ทำ SKU ถัดไป**
   - ย้ายไปหา SKU ถัดไปในรายการ
   - ทำซ้ำ Step 4-5

**💡 เคล็ดลับ:**
- วางแผนเส้นทางหยิบของ → เดินจาก Shelf A → B → C เพื่อประหยัดเวลา
- ตรวจสอบของทุกครั้งก่อนหยิบ → ว่าเป็น SKU ที่ถูกต้องหรือไม่
- ถ้า SKU หลายตัวอยู่ Shelf เดียวกัน → หยิบรวมกันเลย

**⚠️ สิ่งที่ต้องระวัง:**
- ❌ **สแกน SKU ผิด** → ระบบจะแจ้งว่า "SKU นี้ไม่อยู่ใน Batch"
  - **วิธีแก้:** สแกนใหม่ให้ถูก
- ❌ **หยิบเกินจำนวน** → ระบบจะไม่ให้บันทึก
  - **วิธีแก้:** นับใหม่ให้ถูก
- ⚠️ **ลืมสแกน SKU** → Progress จะไม่อัพเดท, Batch ไม่เสร็จ
  - **วิธีแก้:** สแกนทุก SKU ที่หยิบ

**🆘 ถ้าเกิดปัญหา:**
- **QR Code สแกนไม่ได้:** พิมพ์ SKU ด้วยตัวเอง
- **หา SKU ไม่เจอ:** ดู Shelf บนใบงาน → ถ้ายังไม่เจอ บอกทีม Online
- **ของอยู่ Shelf ผิด:** บันทึกตำแหน่งจริง → บอกทีม Online ให้อัพเดท

---

#### 🔹 Step 3: บันทึก Shortage (Mark Shortage)

**เวลาที่ทำ:** เมื่อหยิบสินค้าไม่ครบ (ของไม่พอ)

**วัตถุประสงค์:** บันทึกว่า SKU ไหนขาดกี่ชิ้น เพื่อให้ทีม Online แก้ไข

**ขั้นตอน:**

1. **กด Mark Shortage**
   - เมื่อพิมพ์จำนวนที่หยิบได้น้อยกว่าที่ต้องการ
   - คลิก `⚠️ Mark Shortage`

2. **ระบุข้อมูล Shortage**
   - **จำนวนที่หยิบได้:** (เช่น 5 ชิ้น)
   - **จำนวนที่ขาด:** (เช่น 5 ชิ้น)
   - **เหตุผล:** เลือกจาก dropdown
     - 📦 **สต็อกหมด** (Out of Stock) → ของหมดจริง
     - 💔 **สินค้าเสียหาย** (Damaged) → ของมีแต่เสีย
     - 📍 **หาไม่เจอ** (Not Found) → ของควรมีแต่หาไม่เจอ
     - 📝 **อื่นๆ** (Other) → ระบุเองใน Note

3. **บันทึก**
   - คลิก `✅ บันทึก Shortage`
   - ระบบจะ:
     - บันทึก Picked Qty = 5 ชิ้น
     - บันทึก Shortage Qty = 5 ชิ้น
     - สร้าง Shortage Record ใน Shortage Queue
     - แจ้งทีม Online ให้รู้

4. **แจ้งทีม Online**
   - โทรหรือส่งข้อความแจ้งว่ามี Shortage
   - ทีม Online จะเข้าจัดการใน Shortage Queue

**💡 เคล็ดลับ:**
- บันทึก Shortage ทันทีเมื่อพบว่าของไม่พอ → อย่ารอให้ปลายวัน
- ใส่เหตุผลให้ชัดเจน → เพื่อให้ทีม Online แก้ไขได้ตรงจุด
- ถ้าหาของไม่เจอ → ลองหาใน Shelf ใกล้เคียงก่อน

**⚠️ สิ่งที่ต้องระวัง:**
- ⚠️ **ลืม Mark Shortage** → Batch จะไม่ถึง 100%, ส่งมอบไม่ได้
  - **วิธีแก้:** กลับไปสแกน SKU นั้นอีกครั้ง → Mark Shortage
- ⚠️ **Mark Shortage ผิด SKU** → ทีม Online จะแก้ไขผิด
  - **วิธีแก้:** แจ้งทีม Online ให้แก้ไข

---

#### 🔹 Step 4: ตรวจสอบ Progress

**เวลาที่ทำ:** ตลอดวัน และก่อนส่งมอบให้ Packer

**วัตถุประสงค์:** ดูว่าหยิบของครบหรือยัง

**ขั้นตอน:**

1. **ดู Batch Detail**
   - เข้าหน้า Batch List → คลิกที่ Batch ที่กำลังทำ
   - หรือในหน้า Scan SKU → ดู Progress Bar ด้านบน

2. **ตรวจสอบ Progress Bar**
   - 🟢 **100%** → เสร็จแล้ว พร้อมส่งมอบให้ Packer
   - 🔵 **75%** → เหลืออีก 25%
   - 🟡 **50%** → ทำไปครึ่งทาง
   - 🟡 **25%** → เพิ่งเริ่ม

3. **ดู SKU Progress Table**
   - ตารางจะแสดง SKU ทั้งหมดและสถานะ:
     - ✅ **Completed** (สีเขียว) → หยิบครบ
     - ⏳ **Picking** (สีเหลือง) → ยังหยิบไม่ครบ
     - ❌ **Shortage** (สีแดง) → มีของขาด

4. **ทำต่อจนครบ 100%**
   - ถ้ายังไม่ครบ → ดู SKU ที่เป็น "Picking" → หยิบต่อ
   - ถ้ามี "Shortage" → แจ้งทีม Online

**💡 เคล็ดลับ:**
- ตรวจสอบ Progress ทุก 1-2 ชั่วโมง
- ถ้าเหลือเวลาไม่มาก แต่ Progress ยังต่ำ → เร่งทำหรือขอความช่วยเหลือ

### 📝 การทำงานแต่ละขั้นตอน (Step-by-Step)

---

#### 🔹 Step 1: ตรวจสอบ Batch ที่พร้อมส่ง

**เวลาที่ทำ:** ก่อนเริ่มแพ็ค (16:00)

**วัตถุประสงค์:** ดูว่ามี Batch ไหนที่พร้อมส่งมอบ (Progress 100%)

**ขั้นตอน:**

1. **เข้าหน้า Batch List**
   - เปิดมือถือ/คอมพิวเตอร์
   - เข้าเมนู `Batch Management` → `รายการ Batch`

2. **กรองเฉพาะ Batch พร้อมส่ง**
   - ดู Progress Bar → หา Batch ที่เป็น 🟢 **100%**
   - หรือใช้ Filter: `Progress = 100%`

3. **ตรวจสอบสถานะส่งมอบ**
   - ดูคอลัมน์ "สถานะส่งมอบ":
     - 🟢 **พร้อมส่ง** → ยังไม่ได้สแกนส่งมอบ (ทำได้)
     - 🟡 **รอยืนยัน** → สร้าง Handover Code แล้ว (รอขนส่งมารับ)
     - ✅ **ส่งแล้ว** → ส่งมอบเรียบร้อยแล้ว (ไม่ต้องทำอะไร)

4. **เลือก Batch ที่จะส่งมอบ**
   - คลิกที่ Batch ID เพื่อดูรายละเอียด

**💡 เคล็ดลับ:**
- ควรส่งมอบ Batch ที่ SLA ใกล้ก่อน (ดู Due Date)
- ถ้ามี Batch หลายตัวพร้อมส่ง → จัดกลุ่มตาม Carrier เพื่อให้ขนส่งมารับพร้อมกัน

---

#### 🔹 Step 2: รับงานจาก Picker

**เวลาที่ทำ:** เมื่อ Picker หยิบเสร็จ (16:00)

**วัตถุประสงค์:** รับสินค้าที่ Picker หยิบเสร็จแล้ว

**ขั้นตอน:**

1. **Picker แจ้งว่า Batch เสร็จ**
   - Picker จะบอกว่า Batch ไหนพร้อมแพ็ค

2. **ตรวจสอบของ**
   - นับว่าครบตามใบงาน Picking List หรือไม่
   - ตรวจสอบว่าของถูกต้อง (SKU ตรงกับออเดอร์)

3. **จัดเตรียมวัสดุแพ็ค**
   - กล่อง, เทป, สติกเกอร์

---

#### 🔹 Step 3: แพ็คสินค้า

**เวลาที่ทำ:** หลังรับงานจาก Picker (16:00-17:00)

**วัตถุประสงค์:** แพ็คสินค้าตามออเดอร์เพื่อส่งมอบให้ขนส่ง

**ขั้นตอน:**

1. **แพ็คตาม Order**
   - แยกของตาม Order Number
   - ใส่กล่อง/ซองตามขนาด

2. **ติดสติกเกอร์/ข้อมูล**
   - ติด Order Number
   - ติด Tracking Number (ถ้ามี)
   - ติดข้อมูลลูกค้า (ชื่อ, ที่อยู่)

3. **จัดกลุ่มตาม Carrier**
   - แยกกลุ่มตาม Carrier (SPX, Flash, LEX, J&T)
   - เพื่อให้ส่งมอบได้ง่าย

**💡 เคล็ดลับ:**
- แพ็คอย่างระมัดระวัง → ของแตกจะทำให้ลูกค้าไม่พอใจ
- ใช้กล่องที่เหมาะสมกับขนาด → ประหยัดค่าส่ง
- ตรวจสอบซ้ำก่อนปิดกล่อง → ว่าของครบหรือไม่

---

#### 🔹 Step 4: สแกนส่งมอบ (Scan Handover)

**เวลาที่ทำ:** หลังแพ็คเสร็จ (17:00)

**วัตถุประสงค์:** บันทึกว่าส่งมอบ Batch นี้ให้ขนส่งแล้ว

**ขั้นตอน:**

1. **เข้าหน้า Scan Handover**
   - เปิดมือถือ/แท็บเล็ต
   - เข้าเมนู `สแกนคลัง` → `ยืนยันส่งมอบ Batch`

2. **สแกน Batch QR Code**
   - **วิธีที่ 1:** คลิก `📷 สแกน Batch` → สแกน QR บนใบงาน
   - **วิธีที่ 2:** พิมพ์ Batch ID ด้วยตัวเอง

3. **ระบบสร้าง Handover Code**
   - ระบบจะสร้าง Handover Code อัตโนมัติ
   - รูปแบบ: `H-YYYYMMDD-NNN` (เช่น H-20251119-001)
   - จะแสดง QR Code ของ Handover Code

4. **บันทึก**
   - คลิก `✅ ยืนยันส่งมอบ`
   - ระบบจะบันทึก:
     - Batch Status = DISPATCHED
     - Handover Timestamp
     - Handover By = คุณ

**💡 เคล็ดลับ:**
- สแกนทันทีหลังแพ็คเสร็จ → เพื่อให้ข้อมูลอัพเดท
- เก็บ Handover Code ไว้ดี → จะใช้ตอนขนส่งมารับ

---

#### 🔹 Step 5: พิมพ์ใบส่งมอบ (Handover Slip)

**เวลาที่ทำ:** หลังสแกนส่งมอบ (17:00)

**วัตถุประสงค์:** พิมพ์เอกสารเพื่อส่งมอบให้ขนส่ง

**ขั้นตอน:**

1. **พิมพ์ใบส่งมอบ**
   - ในหน้า Scan Handover → คลิก `🖨️ พิมพ์ใบส่งมอบ`
   - หรือไปที่ `Batch Detail` → คลิก `พิมพ์ใบส่งมอบ`

2. **เอกสารจะมี:**
   - Batch ID
   - Handover Code + QR Code
   - รายการออเดอร์
   - จำนวนกล่อง/ชิ้น
   - วันที่ส่งมอบ
   - ผู้ส่งมอบ (คุณ)

3. **เซ็นเอกสาร**
   - เซ็นชื่อผู้ส่งมอบ
   - พร้อมให้ขนส่งเซ็นรับ

**💡 เคล็ดลับ:**
- พิมพ์ 2 ชุด → 1 ชุดให้ขนส่ง, 1 ชุดเก็บไว้เป็นหลักฐาน

---

#### 🔹 Step 6: เรียกขนส่งมารับ

**เวลาที่ทำ:** หลังพิมพ์ใบส่งมอบ (17:00-18:00)

**วัตถุประสงค์:** ติดต่อขนส่งให้มารับของ

**ขั้นตอน:**

1. **ติดต่อขนส่ง**
   - โทรหรือใช้แอปขนส่ง
   - แจ้งจำนวนกล่อง/ชิ้น

2. **เตรียมของ**
   - จัดกลุ่มตาม Carrier
   - เตรียมใบส่งมอบ

3. **ส่งมอบ**
   - ให้ขนส่งนับของ
   - ให้ขนส่งเซ็นรับในใบส่งมอบ
   - เก็บใบส่งมอบที่ขนส่งเซ็นแล้ว

**💡 เคล็ดลับ:**
- ให้ขนส่งนับของหน้าตา → เพื่อป้องกันของหาย
- เก็บใบส่งมอบไว้อย่างน้อย 1 เดือน

| **ส่งมอบ Batch ที่ Progress < 100%** | ของไม่ครบ, ลูกค้าร้องเรียน | ตรวจสอบ Progress ก่อนส่งมอบ |
| **ไม่เก็บใบส่งมอบ** | ไม่มีหลักฐานว่าส่งแล้ว | เก็บทุกใบ |

### ✅ Checklist ก่อนเลิกงาน (ทีม Packer)

**ก่อนเลิกงานทุกวัน ให้ตรวจสอบ:**

- [ ] **ส่งมอบ Batch ทั้งหมดให้ขนส่งแล้วหรือยัง?**
  - ตรวจสอบ Batch List → ดูว่ามี Batch ที่ Progress 100% แต่ยังไม่ส่งมอบ

- [ ] **พิมพ์ใบส่งมอบครบหรือยัง?**
  - นับใบส่งมอบ → ต้องตรงกับจำนวน Batch ที่ส่ง

- [ ] **ขนส่งเซ็นรับครบหรือยัง?**
  - ตรวจสอบใบส่งมอบ → ต้องมีลายเซ็นขนส่ง

- [ ] **เก็บใบส่งมอบไว้ดีหรือยัง?**
  - จัดเก็บในแฟ้มประจำวัน

- [ ] **มีของค้างแพ็คหรือไม่?**
  - ถ้ามี → แพ็คให้เสร็จหรือส่งต่อให้เพื่อน

## 🤝 Teamwork & Communication Tips

### การทำงานเป็นทีม

**1. การสื่อสารระหว่างทีม**

✅ **DO (ควรทำ):**
- แจ้งปัญหาทันที → อย่ารอให้ปลายวัน
- ใช้ช่องทางสื่อสารที่ตกลงกัน (Line Group, Walkie-Talkie)
- แจ้งเมื่อเสร็จงาน → เพื่อให้ทีมถัดไปรู้
- ถามเมื่อไม่เข้าใจ → อย่าเดาเอง

❌ **DON'T (ไม่ควรทำ):**
- ไม่แจ้งปัญหา → ทำให้งานล่าช้า
- ใช้ช่องทางส่วนตัว → ข้อมูลหาย
- ทำผิดแล้วไม่บอก → ปัญหาซ้อนทับ

**2. การประสานงานระหว่างแผนก**

**ทีม Online → Picker:**
- ส่งใบงาน Picking List พร้อม Batch
- แจ้งเมื่อมี Batch ใหม่
- แจ้งเมื่อ Resolve Shortage แล้ว

**Picker → Online:**
- แจ้งเมื่อรับงาน Batch
- แจ้งเมื่อมี Shortage
- แจ้งเมื่อหยิบเสร็จ 100%

**Picker → Packer:**
- ส่งมอบของพร้อมใบงาน
- แจ้งถ้ามี Shortage ที่ลูกค้าต้องรู้
- แจ้งถ้ามีของพิเศษที่ต้องดูแลพิเศษ

**Packer → Online:**
- แจ้งเมื่อส่งมอบให้ขนส่งแล้ว
- แจ้งถ้าขนส่งปฏิเสธรับของ

**3. การช่วยเหลือกัน**

- ถ้าทีมหนึ่งงานเยอะ → ทีมอื่นช่วย (ถ้าทำได้)
- ถ้า Picker คนหนึ่งรับ Batch หนัก → แบ่งกันหยิบ
- ถ้า Packer แพ็คไม่ทัน → Picker ช่วยแพ็ค

**4. การแก้ไขปัญหาร่วมกัน**

**Scenario: สินค้าขาดเยอะ**
- Picker: แจ้งทีม Online ทันที
- Online: เช็คสต็อก → ถ้ามีผิดที่ Import → แก้ไข
- Picker: ลองหาของใน Shelf อื่น
- ถ้าไม่มีจริง → Mark Shortage → Online Resolve

**Scenario: Batch ต้องส่งด่วน แต่ Progress ยังต่ำ**
- Online: แจ้ง Picker ว่าต้องเร่งด่วน
- Picker: หยิบ Batch นี้ก่อน, ขอความช่วยเหลือจากเพื่อน
- Packer: เตรียมวัสดุแพ็คล่วงหน้า

### Best Practices (แนวทางปฏิบัติที่ดี)

**1. ความรับผิดชอบ**
- ทำงานที่ได้รับมอบหมายให้เสร็จ
- ไม่ทิ้งงานค้างให้เพื่อน
- ตรวจสอบงานก่อนส่งต่อ

**2. ความรวดเร็ว**
- ทำงานอย่างรวดเร็ว แต่ไม่เร่งรีบจนผิดพลาด
- วางแผนเส้นทางหยิบของ → ประหยัดเวลา
- ใช้ Quick Assign และ Shortcut ในระบบ

**3. ความแม่นยำ**
- ตรวจสอบทุกครั้งก่อนสแกน
- นับของให้ถูกต้อง
- อ่านใบงานให้ละเอียด

**4. ความสะอาด**
- เก็บของกลับที่เดิม
- จัดระเบียบคลัง
- ไม่ทิ้งขยะ

**5. การเรียนรู้**
- ถามเมื่อไม่เข้าใจ
- สอนเพื่อนใหม่
- แชร์ Tips ที่ดี

## 🆘 การแก้ปัญหาเบื้องต้น (Troubleshooting)

### ปัญหาที่เจอบ่อย + วิธีแก้

---

#### 📱 ปัญหาระบบและการสแกน

**Q1: สแกน QR Code ไม่ได้ / กล้องไม่ทำงาน**

**สาเหตุ:**
- กล้องไม่มีสิทธิ์เข้าถึง
- QR Code เสียหาย/พิมพ์ไม่ชัด
- แสงสว่างไม่พอ

**วิธีแก้:**
1. ให้สิทธิ์กล้องในการตั้งค่ามือถือ
2. ถ้า QR Code เสีย → พิมพ์ใหม่
3. ถ้าสแกนไม่ได้ → พิมพ์ Batch ID / SKU ด้วยตัวเอง

---

**Q2: ระบบช้า / หน้าเว็บโหลดนาน**

**สาเหตุ:**
- อินเทอร์เน็ตช้า
- เปิดหลายแท็บ
- ข้อมูลเยอะ

**วิธีแก้:**
1. Refresh หน้า (F5)
2. ปิดแท็บอื่น
3. ถ้ายังช้า → รอสักครู่แล้วลองใหม่
4. ถ้ายังช้ามาก → แจ้ง Admin

---

**Q3: Login ไม่ได้ / ลืมรหัสผ่าน**

**สาเหตุ:**
- พิมพ์ Username/Password ผิด
- ลืมรหัสผ่าน

**วิธีแก้:**
1. ตรวจสอบ Username และ Password
2. ตรวจสอบ Caps Lock
3. ถ้าลืมรหัส → แจ้ง Admin ให้รีเซ็ต

---

#### 📦 ปัญหาการ Import ข้อมูล (ทีม Online)

**Q4: Import Orders แล้วขึ้น Error**

**Error: "Missing required columns"**
- **สาเหตุ:** ไฟล์ Excel ขาดคอลัมน์ที่จำเป็น
- **วิธีแก้:** เช็คว่ามีคอลัมน์ `Order Number`, `Platform`, `SKU`, `Quantity`, `Shop Name` หรือไม่

**Error: "SKU not found"**
- **สาเหตุ:** SKU ไม่มีในฐานข้อมูล
- **วิธีแก้:** Import Products ก่อน → แล้วค่อย Import Orders

**Error: "Duplicate order"**
- **สาเหตุ:** Order Number ซ้ำ
- **วิธีแก้:** ลบออเดอร์เก่า → Import ใหม่

---

**Q5: Import แล้วออเดอร์ไม่แสดงใน Dashboard**

**สาเหตุ:**
- ออเดอร์ถูก classify เป็น SHORTAGE
- Platform ผิด

**วิธีแก้:**
1. ดู Dashboard → กรองตาม Status = "All"
2. ถ้าเจอ → ดูว่าเป็น SHORTAGE หรือไม่
3. ถ้าเป็น SHORTAGE → Import Stock → ออเดอร์จะเปลี่ยนเป็น READY

---

#### 📦 ปัญหาการสร้าง Batch (ทีม Online)

**Q6: สร้าง Batch แล้วไม่มีออเดอร์**

**สาเหตุ:**
- กรอง Platform / Date ผิด
- ออเดอร์ทั้งหมดเป็น SHORTAGE
- ออเดอร์อยู่ใน Batch อื่นแล้ว

**วิธีแก้:**
1. เช็คเงื่อนไข Platform และ Date
2. ดู Dashboard → ดูว่ามีออเดอร์ที่เป็น READY หรือไม่
3. ถ้าไม่มี → Import Orders ใหม่

---

**Q7: สร้าง Batch ผิด Platform / Date**

**วิธีแก้:**
- ลบ Batch (Admin เท่านั้น) → สร้างใหม่
- ถ้าไม่ใช่ Admin → แจ้ง Admin ให้ลบ

---

#### 🔍 ปัญหาการหยิบสินค้า (ทีม Picker)

**Q8: สแกน SKU แล้วขึ้น "SKU not in this Batch"**

**สาเหตุ:**
- สแกน SKU ผิด → ไม่ใช่ของ Batch นี้
- สแกน SKU ที่หยิบเสร็จแล้ว

**วิธีแก้:**
1. ดูใบงาน Picking List → ตรวจสอบว่า SKU นี้อยู่ใน Batch หรือไม่
2. ถ้าไม่อยู่ → สแกน SKU ที่ถูก
3. ถ้าหยิบเสร็จแล้ว → ข้ามไป SKU ถัดไป

---

**Q9: หาของไม่เจอ / Shelf ว่างเปล่า**

**สาเหตุ:**
- ของอยู่ Shelf ผิด
- ของหมดแล้ว แต่ระบบไม่อัพเดท

**วิธีแก้:**
1. ลองหาใน Shelf ใกล้เคียง
2. ถามเพื่อน → อาจเห็นของอยู่ที่อื่น
3. ถ้าหาไม่เจอ → Mark Shortage → แจ้งทีม Online

---

**Q10: หยิบของแล้วไม่รู้ว่าหยิบไปกี่ชิ้น**

**วิธีแก้:**
1. นับใหม่
2. ถ้านับไม่ได้ → ประมาณจำนวน → บันทึก → ถ้าผิดจะมี Shortage

---

#### 📋 ปัญหาการแพ็คและส่งมอบ (ทีม Packer)

**Q11: Batch ยังไม่ถึง 100% แต่ต้องส่งด่วน**

**สาเหตุ:**
- Picker ยังหยิบไม่เสร็จ
- มี Shortage ที่ยังไม่ Resolve

**วิธีแก้:**
1. ติดตามทีม Picker → ถามว่าเสร็จเมื่อไหร่
2. ถ้ามี Shortage → แจ้งทีม Online ให้ Resolve
3. ถ้าเร่งด่วนมาก → แยก Batch (ต้องให้ Admin ทำ)

---

**Q12: ลืมสแกนส่งมอบ / สแกนหลังขนส่งไปแล้ว**

**วิธีแก้:**
- สแกนย้อนหลัง → ระบบจะบันทึกเวลาปัจจุบัน
- แจ้ง Admin ถ้าต้องการแก้เวลา

---

**Q13: ขนส่งปฏิเสธรับของ**

**สาเหตุ:**
- ของไม่ครบ
- ของไม่ตรงกับใบงาน
- ของเสียหาย

**วิธีแก้:**
1. ตรวจสอบของ → แก้ไขให้ถูกต้อง
2. ถ้าของไม่ครบ → ตรวจสอบ Progress
3. ถ้าของเสียหาย → แพ็คใหม่

---

#### ⚙️ ปัญหาอื่นๆ

**Q14: ไฟดับ / ระบบล่ม**

**วิธีแก้:**
1. รอให้ไฟกลับ
2. Login ใหม่
3. ข้อมูลจะไม่หาย → ระบบบันทึกทุกครั้งที่สแกน

---

**Q15: พิมพ์เตอร์เสีย / พิมพ์ไม่ออก**

**วิธีแก้:**
1. เช็คกระดาษ / หมึก
2. Restart พิมพ์เตอร์
3. ใช้พิมพ์เตอร์ตัวอื่น
4. ถ้าพิมพ์ไม่ได้จริงๆ → แสดง QR บนหน้าจอให้สแกน

---

**Q16: ข้อมูลไม่ตรงกับความเป็นจริง (เช่น สต็อกไม่ถูกต้อง)**

**สาเหตุ:**
- ข้อมูล Import ผิด
- มีคนหยิบแล้วไม่สแกน
- นับสต็อกผิด

**วิธีแก้:**
1. แจ้งทีม Online
2. ทีม Online → Import Stock ใหม่ที่ถูกต้อง
3. นับสต็อกจริง → อัพเดทระบบ

### คำถามที่พบบ่อย (FAQ)

**Q: ระบบใช้ได้บนมือถือไหม?**
A: ได้ครับ โดยเฉพาะหน้าสแกน (Scan Batch, Scan SKU, Scan Handover) ออกแบบมาสำหรับมือถือโดยเฉพาะ

**Q: ต้องติดตั้งแอปหรือไม่?**
A: ไม่ต้องครับ เปิดผ่าน Browser (Chrome, Safari) บนมือถือหรือคอมพิวเตอร์ได้เลย

**Q: ข้อมูลจะหายถ้าปิดหน้าเว็บไหม?**
A: ไม่หายครับ ข้อมูลบันทึกในฐานข้อมูล → Login ใหม่จะเจอข้อมูลเดิม

**Q: สามารถแก้ไขข้อมูลหลัง Import ได้ไหม?**
A: ขึ้นอยู่กับ Role:
- **Admin:** แก้ไขได้ทุกอย่าง
- **Staff/Picker/Packer:** แก้ไขบางอย่างได้ (เช่น Mark Shortage)

**Q: ถ้า Import ข้อมูลผิด ต้องทำยังไง?**
A: ลบข้อมูลที่ผิด (Admin) → Import ใหม่

**Q: Batch สามารถสร้างล่วงหน้าได้ไหม?**
A: ได้ครับ สร้างได้ตลอด → แต่ควรสร้างก่อน 1 วัน เพื่อให้ทีมคลังมีเวลาเตรียมตัว
